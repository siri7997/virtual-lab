MD5 Algorithm

Aim Objectives Procedure Theory Simulation

Aim

The primary aim of the MD5 algorithm is to provide a secure way to verify data integrity by generating a unique digital fingerprint (hash value) for any given input data. This ensures that any alterations to the data can be detected.

Objectives

Data Integrity: Ensure that data has not been tampered with during transmission or storage.
Authentication: Verify the authenticity of data by comparing expected and actual hash values.
Non-repudiation: Prevent denial of involvement in data creation or transmission.
Efficiency: Provide a fast and efficient method for generating hash values.

Procedure

Append Padding Bits


The message is padded to make its length 448 mod 512. A 1 bit (0x80) is added, followed by 0s until the length is 448 mod 512.

Append Length

The original length of the message (in bits) is appended as a 64-bit value. This ensures the total message length is exactly a multiple of 512 bits. Initialize MD Buffer

Four 32-bit registers are initialized as follows:

A = 0x67452301 B = 0xEFCDAB89 C = 0x98BADCFE D = 0x10325476

Process Message in 512-bit Blocks

The message is divided into 512-bit (64-byte) blocks. Each block is processed independently. Process Each Block with MD5 Rounds
Each block undergoes 4 rounds of processing (64 operations in total). Each round applies different bitwise operations and modular additions. Update Registers (A, B, C, D)
The values of A, B, C, and D are modified during processing. After all rounds, these registers are added to the original values.

Compute Final MD5 Hash

The final hash is formed by concatenating the hexadecimal values of A, B, C, and D. The result is a 128-bit hash value (32 hexadecimal characters).
