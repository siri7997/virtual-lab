PolyAlphabetic Cipher

Aim Objectives Procedure Theory Simulation

Aim

The primary aim of a poly alphabetic cipher is to enhance the security of encrypted messages by using multiple substitution alphabets, thereby disguising the true frequency of plaintext letters and making cryptanalysis more difficult.

Objectives

Enhanced Security: To provide a higher level of security compared to mono alphabetic ciphers by using multiple substitution alphabets.
Resistance to Frequency Analysis: To disguise the frequency of plaintext letters, thus preventing attackers from using frequency analysis to break the code.
Key-Based Encryption: To use a key (or keyword) to control the selection of substitution alphabets during encryption and decryption.
Algorithm Development: To create and implement algorithms that can efficiently switch between different substitution alphabets based on the key.
Cryptanalysis Prevention: To develop ciphers that are resistant to common cryptanalytic techniques like the Kasiski method and index of coincidence.

Procedure

Key Selection:

Choose a keyword or key phrase.

Alphabet Selection:

Determine the number of alphabets to be used. The Vigenere cipher, for example, uses 26 alphabets, each shifted by one position.

Encryption:

Repeat the keyword so that it matches the length of the plaintext.
For each letter in the plaintext, use the corresponding letter in the repeated keyword to select a substitution alphabet.
Encrypt the plaintext letter using the selected substitution alphabet.

Decryption:

Repeat the keyword so that it matches the length of the cipher text.
For each letter in the cipher text, use the corresponding letter in the repeated keyword to determine the substitution alphabet that was used for encryption.
Decrypt the cipher text letter using the determined substitution alphabet.
